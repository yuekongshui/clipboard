# Clipboard

[English](README.en.md) | [简体中文](README.md) | 日本語

**Swift + SwiftUI + SwiftData** に基づく、ローカルの macOS 用クリップボード管理アプリケーションです。

## 主な機能

* **複数タイプのサポート**：テキスト、画像、ファイルなど、複数タイプのコンテンツ管理をサポートします。
* **履歴管理**：システムクリップボードの変更を自動的に監視し、履歴を保存して、再コピーおよび使用をサポートします。
* **高速検索とフィルタリング**：キーワード検索、およびコンテンツタイプ、カテゴリ、お気に入りステータスによる多次元の組み合わせフィルタリングをサポートします。
* **カテゴリとお気に入り**：カスタムカテゴリ管理（作成、編集、削除）をサポートし、頻繁に使用する記録をお気に入りとしてマークできます。
* **プライバシー保護**：機密コンテンツ（パスワード、トークン、確認コードなど）の自動フィルタリングと、アプリケーションのブラックリスト設定をサポートします。

## 技術スタック

* **プラットフォーム**：macOS
* **開発言語**：Swift
* **UI フレームワーク**：SwiftUI
* **データ永続化**：SwiftData

## アーキテクチャ設計

プロジェクトは明確な階層型アーキテクチャ（MVVM + Service）を採用しています。
* **Models**：データモデル層。クリップボード記録（`ClipboardItem`）、カテゴリ（`Category`）、およびグローバル設定（`AppSettings`）が含まれます。
* **Services**：ビジネスロジック層。クリップボード監視（`ClipboardMonitorService`）、データ永続化（`ClipboardStorageService`）、ファイルストレージ（`FileStorageService`）、およびプライバシーフィルタリング（`PrivacyFilterService`）が含まれます。
* **ViewModels**：ビューモデル層。状態管理とビューロジック処理を担当します。
* **Views**：ビュー層。macOS ネイティブの3カラム構造（`NavigationSplitView`）で設計されています。

## データのプライバシーとセキュリティ

* **完全ローカル**：すべての履歴記録とビジネスデータはローカルデバイスにのみ保存され、ネットワークへの依存やクラウド同期の動作は一切ありません。
* **機密情報の保護**：疑わしい機密テキストに対しては、デフォルトで「保存せずにブロックする」という保守的な戦略が採用されています。
* **大容量ファイルストレージの最適化**：データベースの肥大化を防ぐため、画像はローカルディレクトリにのみ保存され、そのパスが記録されます。ファイルは参照パスのみが保存され、物理ファイルはコピーされません。
